作业案例

  • 1、创建一个用户，用户名为keystone23，密码为000000，邮箱为keystone23@qq.com，域为admin

openstack user create keystone23 --password 000000 --email keystone23@qq.com --domain admin

  • 2、创建一个项目，项目名为project23，域为admin

openstack project create project23 --domain admin

  • 3、创建一个角色，角色名为lsc

openstack role create lsc

  • 4、把用户Keystone23添加一个项目，项目名为project23,角色名为lsc

openstack role add --user keystone23 --project project23 lsc

  • 6、列出所有镜像

openstack image list

  • 7、上传两个镜像，cirros23和centos23 
openstack image create "cirros23" --disk-format qcow2 --container-format bare --file /opt/cirros-0.5.2-x86_64-disk.img
openstack image create "centos23" --disk-format qcow2 --container-format bare --file /opt/images/CentOS_7.2_x86_64_XD.qcow2
  • 8、设置centos23镜像为公共，不可修改的

openstack image set centos23 --protected --public

  • 9、创建一个用户，并给这个用户分配项目和角色

    openstack user create test23 --password 000000 --domain admin 
openstack role add --user test23 --project admin admin
  • 10、创建一个虚拟主机类型flavor-create <虚拟主机名> <1024m> <1g> <1cpu数> <不是公共的> 
nova flavor-create flavor23 23 1024 2 1 --is-public False

openstack flavor list

openstack flavor show flavor23

把虚拟主机flavor23放入到id为a2e6d072dd65408d98411f8b4a524552的项目中

nova flavor-access-add flavor23 a2e6d072dd65408d98411f8b4a524552
  • 11、创建一个安全组，说明为 nothing
openstack security group create group23 --description nothing

展示group23的详细信息

openstack security group show group23
  • 12、修改group23安全组的说明为 test23
openstack security group set group23 --description test23

给安全组group23创建规则，源地址0.0.0.0/0,允许的端口为80，数据包协议为tcp

openstack security group rule create group23 --src-ip 0.0.0.0/0 --dst-port 80 --proto tcp

openstack security group rule list

删除ID为040d0c29-38e8-4d96-b70c-55e5973c3fec的安全组规则


openstack security group rule delete 040d0c29-38e8-4d96-b70c-55e5973c3fec

openstack security group rule create group23 --src-ip 0.0.0.0/0 --dst-port -1 --proto icmp

openstack security group rule list
