docker

docker

  • 查看系统已经存在的镜像包
docker ps
  • 创建容器
docker [run]
-t 分配终端
-i  以交互模式运行容器
--name 容器名字
-p <主机端口:对应容器端口>
-v <路径>:<路径> 主机上的文件挂载到容器内，只写一个路径将会在宿主机上随机找一个路径
--volumes-from <容器名> 用别的容器的卷组
--restart=<always>  退出容器重启

--mount type=volume,source=<卷组名>,target=<挂载在容器中的目录>
<ubuntu> 运行的镜像的名称
</bin/bash> 运行的程序
  • 查看有的容器
docker ps -a
  • 查看启动的容器
docker ps -l
  • 只看容器的ID号
docker ps -q <容器名>
  • 查看容器的运行的进程信息
docker top <进程ID的前4位>
  • 恢复容器运行
docker start <容器ID前4位>
  • 进入容器
docker attach <容器ID前4位>
  • 查看docker容器或者镜像的一些内部信息
docker inspect <容器ID前4位>
  • 删除容器
docker rm 
-v 删除容器时顺便删除卷组
<容器ID前4位>
  • 强制删除
docker rm -f <容器ID前4位>
  • 删除镜像
docker rmi <镜像名>
  • 搜索镜像
docker search <关键字>
  • 拉取镜像
docker pull <镜像名>
  • 查看本地仓库
docker images
  • 创建一个镜像
docker build -t <文件夹名/bdqn> .
  • 查看系统的本质!!!
echo $HOSTNAME
  • 推出仓库
docker push <容器ID>
  • 仓库镜像导入
docker load < <tar包>
  • 打标签
docker tag <容器ID/容器名> <修改后的容器名>
  • 更好的进入容器
docker exec -it <容器ID> /bin/bash
  • 复制容器文件到本机
docker cp <容器ID>:<文件路径> <本地路径>

docker run -d -v
  • 在容器里查看卷组
df
